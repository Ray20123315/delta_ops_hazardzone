# Delta Ops: Hazard Zone — 功能總覽與使用說明

> 本文件為開發中模組的「目前已實作功能」與「管理／使用指引」。適合伺服器管理員與開發者快速上手測試。

**路徑與版本**
- 專案根目錄（workspace）：此專案使用 Minecraft 1.20.1 + Forge 47.4.20、Java 17、Gradle。
- 設定檔與資料：`config/delta_ops_hazardzone/`（包含 `locations.json`、`zones.json`、`item_qualities.json` 等）。

**目前已實作（概要）**
- 伺服器端：
  - 位置資料庫（`LocationDatabase`）
    - 以 `config/delta_ops_hazardzone/locations.json` 儲存：全域大廳出生點（GlobalLobby）與地圖出生點清單（MapSpawns）。
    - API：`setLobby(...)`、`addSpawnPoint(...)`、`getRandomSpawnPoint(...)`、`teleportToLobby(...)`、`listSpawnPoints(...)`、`removeSpawnPoint(...)`。
    - 檔案 I/O 使用 `ReentrantReadWriteLock` 保證執行緒安全。 
  - 管理指令（僅 OP）：`LocationCommand`（註冊在 `/dt` 子命令下）
    - `/dt lobby set`：設定當前位置為大廳出生點（會寫入 `locations.json`）。
    - `/dt spawn add <mapName>`：將當前位置加入指定地圖的出生點清單。
    - `/dt spawn list <mapName>`：列出指定地圖所有出生點（含索引）。
    - `/dt spawn remove <mapName> <index>`：移除指定索引的出生點。
  - 大廳／小隊系統擴充：
    - `LobbySquadManager` 支援輕量化小隊（建立、加入、離隊、隊長轉讓等）。
    - 新增 PendingRequest 與 `handlePlayerTargeting(sender, target)` 智慧路由：若目標無隊伍則為邀請，若有隊伍則視為申請並通知隊長。
  - 配對／發車：`MatchmakingEngine`（存在）可被擴充呼叫 `LocationDatabase.getRandomSpawnPoint(mapName)` 並呼叫 `ServerPlayer.teleportTo(...)` 將整隊玩家傳送到出生點。
  - Zone 與容器：`ZoneSelectionManager` 與 `LootContainerSpawner.spawnFromZones(...)`（可由系統呼叫以根據 zones.json 產生容器）

- 網路（封包）框架：
  - 小隊互動封包：
    - `ServerboundSquadActionPacket`（Client→Server）：主面板操作（LEAVE、INVITE、KICK、TRANSFER）
    - `ClientboundShowRequestScreenPacket`（Server→Client）：指示客戶端顯示接受/拒絕彈窗（包含發起者 UUID 與名稱）
    - `ServerboundHandleRequestPacket`（Client→Server）：玩家在彈窗選擇後回報（Accept / Decline）
    - `ClientboundOpenSquadScreenPacket`（Server→Client）：伺服器指示目標開啟小隊主面板（對應 `/dt menu`）
  - 注意：封包透過模組的 `ModNetwork`（`com.deltaops.network.ModNetwork`）註冊與發送（已於代碼中採用 `PacketDistributor.PLAYER` 方式發送）。請確認在 `ModNetwork.register()` 中完成所有封包註冊。

- 客戶端 GUI（client-only）
  - `SquadMainScreen`：由 `/dt menu` 觸發（目前採最小兼容實作，標註 `@Mod.EventBusSubscriber(value = Dist.CLIENT)`）。
  - `SquadRequestScreen`：伺服器強制彈窗（接收 `ClientboundShowRequestScreenPacket`），目前保留最小實作。
  - 視覺化設計：所有染料圖示皆為純視覺渲染，UI 不佔用真實背包格（目前仍可在下一步實作更完整的按鈕與同步）。

## 快速安裝與建置
1. 安裝 JDK 17。
2. 在專案根目錄執行：

```bash
# Windows
./gradlew.bat build
# Linux / macOS
./gradlew build
```

3. 完成後於 `build/libs/` 取得 jar 並放入伺服器或單人遊戲的 `mods/`。

## 指令與使用（管理者）
- 權限需求：所有會寫入設定/儲存資料的指令皆需 OP 權限（`requires(src -> src.hasPermission(2))`）。

- 大廳與出生點管理（OP）：
  - ` /dt lobby set`  
    - 功能：以執行者當前世界座標、朝向、與所在維度記錄為全域大廳出生點。
    - 備註：會寫入 `config/delta_ops_hazardzone/locations.json`。

  - `/dt spawn add <mapName>`
    - 功能：將執行者當前位置加入 `mapName` 的出生點清單。

  - `/dt spawn list <mapName>`
    - 功能：列出 `mapName` 的所有出生點並顯示索引，用於後續刪除或檢視。

  - `/dt spawn remove <mapName> <index>`
    - 功能：移除指定索引的出生點並儲存。

- 小隊/面板（玩家）
  - `/dt menu`：開啟小隊主面板（客戶端畫面由伺服器指示開啟）。
    - 主面板上的操作會以 `ServerboundSquadActionPacket` 發送至伺服器（邀請、離隊等）。

- 其他（OP）：
  - `/dt zone pos1/pos2/create <name> <priority>`：區域選取與存檔（若已載入 `ZoneCommand`）。
  - `/c`（秘密管理）: 切換執行者於創造 / 生存模式（需 OP），伺服器每 tick 對非白名單創造者強制回生存（由 `SecretCreativeCommand` 實作）。

## 管理介面與流程（小隊邀請範例）
1. 玩家 A 在遊戲中對玩家 B 使用 UI 的邀請操作或在伺服器輸入 `/dt menu` 開啟面板並點選邀請。
2. 伺服器建立 `PendingRequest` 並傳送 `ClientboundShowRequestScreenPacket` 給目標（攜帶發起者 UUID 與名稱）。
3. 目標玩家看到強制彈窗，點選綠染料接受或紅染料拒絕（在可用完整 UI 時會送出 `ServerboundHandleRequestPacket`）。
4. 伺服器接到接受後，將玩家加入對應小隊（以 `LobbySquadManager.joinSquad(...)` 實作加入流程）。

## Matchmaking（發車）整合說明
- 在 `MatchmakingEngine.launchMatch(...)` 呼叫位置建議：
  - 於選定的地圖名稱（`mapName`）中取得隨機出生點：

```java
Optional<LocationDatabase.SpawnPoint> sp = LocationDatabase.getRandomSpawnPoint(mapName);
if (sp.isPresent()) {
    LocationDatabase.SpawnPoint chosen = sp.get();
    for (ServerPlayer p : players) {
        p.teleportTo(targetLevel, chosen.x, chosen.y, chosen.z, chosen.yaw, chosen.pitch);
    }
}
```

- 若要以 zone 驅動容器/出生點分布，可在配對開始或世界啟動時呼叫 `LootContainerSpawner.spawnFromZones(serverLevel);`。

## 檔案位置（重要）
- 設定資料：`config/delta_ops_hazardzone/locations.json`（由 `LocationDatabase` 管理）

- 物品品質：`config/delta_ops_hazardzone/item_qualities.json`（由 `GlobalLootDatabase` 管理）
