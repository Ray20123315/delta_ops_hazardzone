# 三角洲行動：烽火地帶

Minecraft 1.20.1 Forge 的硬核撤離射擊模組。

這個專案旨在把「搜刮、戰鬥、撤離、醫療、死亡掉落、隊伍協作」這一套硬核戰術循環，搬進 Minecraft 中，並以可擴充的資料驅動方式作為後續功能擴展的基礎。

## 模組定位

Delta Ops: Hazard Zone 目前是一個以「戰術撤離」為核心的多人互動模組，重點不是單純的武器與戰鬥，而是把以下流程串成一個完整的高壓任務循環：

- 進入戰區後搜刮資源
- 依據戰鬥價值判斷是否能進入/留在匹配區
- 在指定撤離點完成撤離倒數
- 透過安全箱保存資源與物品
- 經由隊伍/大廳系統進行隊伍管理與回合流程

## 使用手冊

### 一、安裝與先決條件
- 將編譯後的 `delta_ops_hazardzone` jar 放入伺服器的 `mods/` 目錄。
- 此模組適用於 Minecraft 1.20.1 Forge 47.4.20，並需要 Java 17。
- 對於多人伺服器，客戶端與伺服器均須安裝此模組。

### 二、基礎玩法概念
Delta Ops: Hazard Zone 是一款以組隊搜刮、裝備門檻、撤離倒數為核心的硬核玩法。玩家需要：

1. 組隊進入大廳等待。
2. 透過隊長設定或選擇地圖與出生點。
3. 全隊達成準備狀態後嘗試發車。
4. 發車前系統會檢查隊員戰備值是否符合地圖門檻。
5. 發車成功後玩家被傳送到指定地圖的隨機出生點。

### 三、常用指令
以下指令多數需要 OP 權限才能使用：

- `/dt lobby set`
  - 在當前位置設定大廳出生點，這是隊伍開始前的集合位置。
- `/dt spawn add <mapName>`
  - 新增目前位置為指定地圖的隨機出生點。
- `/dt spawn list <mapName>`
  - 顯示指定地圖的所有已設定出生點與索引。
- `/dt spawn remove <mapName> <index>`
  - 移除指定地圖的某一個出生點。
- `/dt menu`
  - 在客戶端打開小隊主面板，便於隊伍管理與發車。

### 四、隊伍系統（Squad）

- 玩家可建立小隊並邀請其他玩家加入。
- 小隊成員可申請加入其他玩家的小隊。
- 隊長可以踢出隊員、移交隊長權限，並切換是否允許自動補員。
- 每名隊員都有「準備」狀態，隊長可由小隊介面或指令觸發發車。

### 五、小隊介面操作
在 `SquadMainScreen` 中，玩家可執行：

- 建立或離開小隊。
- 邀請指定玩家加入。
- 踢出隊員。
- 轉移隊長權限。
- 切換自動配對功能。
- 切換準備狀態。

`SquadRequestScreen` 則會在收到邀請或申請時彈出，玩家可以接受或拒絕該操作。

### 六、地圖與戰備值要求
目前模組已實作地圖與門檻機制，支援以下地圖 ID：

- `zero_dam`：零號大壩，門檻 0。
- `longbow_valley`：長弓溪谷，門檻 30000。
- `space_base_confidential`：航天基地（機密），門檻 187500。
- `space_base_top_secret`：航天基地（絕密），門檻 600000。

發車前系統會依據小隊選擇的地圖，檢查每位隊員的裝備戰備值。
如果隊員的戰備值低於地圖門檻，發車會被取消，且：

- 隊長會收到失敗通知與低戰備玩家資訊。
- 該隊員會收到「你的戰備值過低」警告。

### 七、如何讓隊伍成功發車

1. 先讓所有隊員加入同一小隊。
2. 進入小隊面板並確認隊長已經設定好目標地圖（或伺服器已經設定好出生點）。
3. 所有隊員切換為「已準備」。
4. 隊長執行發車。
5. 若有成員未準備或戰備值未達標，系統會顯示具體錯誤訊息。

### 八、搜刮與物資生成

- 目前模組已支援固定高價值錨點生成，管理員可以在地圖中標記特定位置，該位置會優先生成指定容器。
- 一般搜尋區域會使用網格化引擎分散生成隨機容器，避免資源過度集中。
- `LootContainerSpawner` 會根據 `zones.json` 的區域優先級生成容器，讓高優先級區域產出更多高價值物品。

### 九、管理員提示

- `LocationDatabase` 用於儲存大廳位置與地圖出生點。
- `ZoneSelectionManager` 用於儲存區域位置與搜尋優先級。
- 若要搭配更完整的地圖配置，請先使用 `/dt lobby set` 與 `/dt spawn add <mapName>` 進行初始化。

### 十、測試與驗證建議

1. 以 OP 身份進入伺服器，站在希望作為大廳的位置輸入：

```bash
/dt lobby set
```

2. 設定測試地圖出生點：

```bash
/dt spawn add zero_dam
/dt spawn add zero_dam
```

3. 檢查 `config/delta_ops_hazardzone/locations.json` 是否存有 `globalLobby` 與 `mapSpawns`。
4. 讓玩家使用 `/dt menu` 開啟小隊介面，並執行邀請、加入、準備、發車測試。

### 常見問題

- Q：為什麼發車失敗？
  - A：可能是小隊中有隊員未切換準備、或是裝備價值低於當前地圖門檻。
- Q：如何移除出生點？
  - A：使用 `/dt spawn remove <mapName> <index>`。
- Q：我看不到小隊介面？
  - A：請先確認模組已正確安裝於伺服器與客戶端，並使用 `/dt menu`。

### 目前已實作的功能
- 小隊系統：建立小隊、離隊、邀請、申請、踢出、隊長移交、切換自動配對、準備/取消準備。
- 請求流程：邀請/申請會透過客戶端彈窗顯示，玩家可選擇接受或拒絕。
- 小隊狀態同步：伺服器廣播隊員名稱與準備狀態給全隊成員。
- 發車匹配：`MatchmakingEngine` 支援小隊即時發車、補員邏輯，以及指定地圖的隨機出生點。
- 地圖門檻：`HazardMapRegistry` 登記地圖與最低戰備值，`GearValueEvaluator` 計算玩家裝備價值。
- 門檻檢查：低於地圖戰備值門檻時會取消發車並通知隊長與該隊員。
- 固定錨點 Loot：`FixedLootManager` 管理手動設定的高價值物品錨點並優先生成。
- 網格生成：`GridDistributionEngine` 實作 AABB 網格化隨機物品分散生成。
- 區域 Loot 生成：`LootContainerSpawner` 根據 `zones.json` 的區域優先級生成容器。
- 管理界面：`LootCommand` 與 `AdminItemTaggingScreen` 提供管理員物品標記與品質設定界面。
- 設定與資料庫：`LocationDatabase` 儲存大廳與地圖出生點，`ZoneSelectionManager` 儲存區域選擇與優先級。
